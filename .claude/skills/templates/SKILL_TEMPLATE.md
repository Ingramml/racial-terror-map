## Quick Start Instructions

1. **Copy this template:**
   ```bash
   cp SKILL_TEMPLATE.md your-skill-name/SKILL.md
   ```

2. **Fill in YAML frontmatter:**
   - Set name
   - Write optimized description (critical for discovery!)
   - Set allowed-tools if restrictions needed
   - Set version to 1.0.0

3. **Complete each section:**
   - Purpose (keep brief)
   - When This Activates (list all triggers)
   - Prerequisites (if any)
   - Steps (detailed workflow)
   - Output (what gets produced)
   - Examples (at least 2)

4. **Add supporting files if needed:**
   - Templates for file generation
   - Checklists for step-by-step workflows
   - Reference for detailed patterns

5. **Test the skill:**
   - Verify activation on triggers
   - Test tool restrictions
   - Check supporting files load
   - Validate output

6. **Deploy:**
   - Commit to appropriate repo
   - Update version as needed
   - Document in changelog

## Description Writing Tips

**Formula:**
```
[Action] for [Context]. Use when [Triggers]. [Ensures/Provides].
```

**Good Example:**
```yaml
description: Enforces phase planning protocol for CA Lobby project. Use when user requests to start a new phase, create implementation plan, or begin major feature development. Ensures master plan consultation and proper documentation structure.
```

**Bad Example:**
```yaml
description: Helps with planning
```

**Checklist for Good Descriptions:**
- [ ] Primary action in first 5 words
- [ ] Specific context (not vague)
- [ ] 3+ trigger phrases
- [ ] Explains what it ensures/prevents
- [ ] 2-3 sentences total
- [ ] Keywords user will naturally say
